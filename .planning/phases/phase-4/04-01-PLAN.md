# Plan 04-01: Jurisdictional Style Ingestion (Gold Standard Extraction)

Automate the extraction of courtroom transcription rules from uploaded style guides (PDF/Text) using LLM-assisted parsing.

## Objectives
- Support multi-jurisdictional tagging for court-specific formatting requirements.
- Extract structured `CVLEngine` rules directly from unstructured policy manuals.
- Provide a reliable ingestion pipeline for administrative auditors.

## Proposed Changes

### Database Schema
- [x] Added `jurisdiction`, `source_url`, and `source_key` to `style_guides` table.
- [x] Ran migrations to update local Postgres.

### API Enhancements
- [x] **File Extraction Utility**: Created `apps/api/src/utils/file-extractor.ts` using `pdf-parse`.
- [x] **Ingestion Logic**: Added `StyleGuideService.ingestFromSource` utilizing Claude 3.5 Sonnet to parse legal manuals into structured JSON rules.
- [x] **Modular Routes**: Added `POST /style-guides/:id/ingest` and updated `POST /style-guides` to handle R2 uploads and jurisdictional metadata.

### Frontend Enhancements
- [x] **Admin Tooling**: Redesigned `StyleGuideAdminPage.tsx` with name/jurisdiction fields and an "Extract Rules" trigger for the automated pipeline.

## Verification Plan

### Automated Tests
- Test PDF text extraction with a sample courtroom manual.
- Verify JSON schema of rules generated by Claude.

### Manual Verification
1. Upload a sample legal style guide PDF.
2. Tag it with a jurisdiction (e.g., "US-FL").
3. Trigger "Extract Rules" and verify that the rules appear in the database for that guide.
4. Verify that the rules are correctly assigned types (filler-removal, slang-normalization, etc.).
